  2. Let's make a raw pagination scheme with buttons:
    • Keep some state about the current page
      • The initial page should be zero;
      • The current page state will be used as an index for the input data.
    • Add two buttons for switching pages
      • Add a button with the btn_next id. When btn_next is clicked, increase the current page value by one and update the views to display data related to the animal found while using the current page as the index;
      • Add a button with the btn_previous id. When btn_previous is clicked, decrease the current page value by one and update views to display data related to the animal found while using the current page as the index.
    • Implement a wrap-around feature on the pagination.
      • If the current page state change is out of the bounds of the input data, then constrain the current page with a wrap-around strategy. If the current page gets one below zero, go to the last valid index on the data and vice-versa in the opposite case.
  3. Use Picasso to load the images
    • Use imageAnimals data as the URL for photos. The provided default values are real links, which will make it possible to retrieve the image from the web on your manual testing;
    • The target image view should be the view with the image_view id;
    • Set a placeholder image using the drawable with the placeholder.xml id;
    • Set an error image using the drawable with the error.xml id.
